# Incremental Strategy for Applying Mutation Operators Emphasizing Faults Difficult to be Detected by Automated Static Analyser
SILVA, Vinícius ; SPOTO, Edmundo; ARAÚJO, Cláudio Antônio; VINCENZI, Auri. Incremental Strategy for Applying Mutation Operators Emphasizing Faults Difficult to be Detected by Automated Static Analyser. SBES‘17: Proceedings of the 31st Brazilian Symposium on Software Engineering, Ceará, Brasil, p. 24-33, 2017. Disponível em: https://dl.acm.org/doi/10.1145/3131151.3131169 . Acesso em: 29 abr. 2021.

## 1.	Fichamento do Conteúdo
O conteúdo abordado no artigo é referente a testes de mutações, o objetivo inicial é fazer testes que rodam automaticamente para testar os códigos. O intuito do documento se dá pelo fato de querer dar continuidade em um dos estudos que já havia sido publicado anteriormente por um dos autores do artigo em questão. Pode-se perceber que ao decorrer do texto, os autores conseguem desmitificar e destacar alguns conceitos e características sobre os testes de mutação, bem como seus principais pontos e suas características mais fracas. Dentro dos conceitos básicos, é destacado o fato de que as validações do software podem ser divididas em dois tipos, sendo eles: atividades de análise dinâmica e atividades de análise estática, é explicado a diferenciação de conceito entre eles. Os autores, destacam todas as ferramentas utilizadas para realização dos testes, sendo uma diferente para gerar os mutantes e outra diferente para fazer a atividades de análise estática automatizadas que neste caso foi usado a Splint. A linguagem escolhida para desenvolvimento desses testes, foi a linguagem C. Ao escolher a ferramenta para trabalharem, os autores mostraram outras ferramentas disponíveis para fazer o mesmo tipo de teste, porem de formas diferentes. O artigo, de modo geral, é bem objetivo e sucinto, com foco de exibição dos resultados e todos os recursos usados, ficando bem claro de entender que o conhecimento e dedicação para o artigo, foi enorme. Pode-se ficar claro, que o objetivo final foi alcançado, mostrar que os testes de mutação podem ser eficientes e que conseguem contornar muito bem os softwares.
